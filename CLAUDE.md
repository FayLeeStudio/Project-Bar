# Sand Together — Claude Code 契约

> 跨 session 硬约束，每次启动必读必守。
> 这里**只放不随方案演进而变的红线** + 指针；会演化的架构 / 协议 / 玩法放 `doc/`，本文件只指过去。
> 不确定某个改动是否违约时，**先提问，不要直接改**。

---

## 这是什么

一个透明、始终置顶的桌面叠加层（"智能贴纸"，Bongo-Cat 式悬浮），把多人实时数字劳动数据可视化为**像素沙粒落入瓶中**：每位参与者的键盘输入化为带颜色的沙粒，在共享的沙井中堆积、沉降、形成地质层。基调是轻松的派对感，是"带数据叙事的工具"。

---

## 不变量（红线，违反前必须先问）

以下是**无论架构怎么调都不动**的约束。具体技术选型——物理在哪算、同步模型、线协议——**不在此列**，属会演进的架构选择，见 `doc/`。

1. **隐私红线（最高优先级）。** 只对输入**计数**，绝不记录具体键位内容、绝不记录输入文本。客户端、服务端、存档、归档带——**任何环节一律不得出现键位内容**。

2. **纯 Web、无构建步骤。** UI 是纯 HTML/CSS/vanilla JS，无前端框架、无构建 / 打包 / 转译工具，`index.html` 保持纯文本可直接编辑、由后端 `GET /` 直接托管（改完 `git pull` 即生效）。允许后端**同源托管额外的纯 JS**（如前后端共享的模拟模块），但**不得**引入任何构建步骤。除非显式批准，此约束不可更改。

3. **UI 与外壳解耦，方向不可反。** `index.html` = 纯 Web，任意设备打开该 URL 即可玩、也可改；Tauri 外壳只**加载**后端托管的那个 URL，不承载 UI 逻辑；UI 不依赖外壳才能在浏览器里跑。

4. **原生边界。** 全局输入统计**只在 Rust 层**做（rdev），webview **永不**直接读全局输入；webview 只通过 Tauri `keycount` 事件接收整数计数。

5. **房间有唯一权威真相。** 每个房间的共享画布 / 冻结地层是**全员一致的单一权威记录**，由**服务端**裁决（输入定序、freeze 仲裁）并负责持久化（无限期保留，全员主动弃用才回收）。客户端**可**本地参与模拟以降延迟，但**不得自成第二权威、不得静默发散**——必须可被服务端的 snapshot / checksum 纠正、以服务端为准对账。
   > ⚠️ "物理具体在哪跑、客户端跑多少、用什么同步协议" 是**会演进的架构选择**（历程：客户端本地物理 → 服务端权威 + 客户端纯渲染 → 当前转向确定性帧同步 / lockstep）。当前方向与契约见 `doc/architecture.md`「同步模型」，**勿把某一代实现写死成不变量**。

## 平台规则

- **Windows 是当前主力开发 / 构建目标**，macOS 之后支持。
- **Tauri 不跨平台编译**：Windows 包在 Windows 上构建，macOS 包在 macOS 上构建（或用 CI）。UI 层（网页）跨平台免费，两端渲染一致。
- macOS 额外项（以后处理）：透明需 `macOSPrivateApi: true`；全局输入需"输入监控"授权。
- **后端是常驻、有状态进程**，部署在海外位置的 VPS（大陆厂商香港 / 新加坡等海外节点亦可），保证大陆与海外都可连；本地开发用 `npm run server` 起 localhost 实例联调。

## 给 Claude Code 的护栏

- **不要**给网页 UI 引入前端框架或构建步骤。
- **不要**把输入统计逻辑搬进 webview（仍只在 Rust / rdev）。
- **不要**在任何环节（客户端 / 服务端 / 存档 / 归档带）存储或传输键位内容——只存计数与网格像素。
- 客户端**可以**镜像服务端做本地模拟，但**不得脱离服务端裁决自成权威**、或维护一套与服务端冲突且**不对账**的状态；必须能被服务端 snapshot / checksum 纠正。
- 改动 `doc/architecture.md`（含「同步模型」）/ `doc/backend.md` 中的技术选型或线协议前，必须先说明理由并等待确认。
