> 会演化的完整方案、阶段计划、代码参考放在 `docs/DESIGN.md`,不要写进这里,保持本文件精简。

## 这是什么

一个透明、始终置顶的桌面叠加层("智能贴纸"),把数字劳动相关数据可视化成**马里奥赛车式的赛道小地图**:同一房间内的多个用户,根据一份共享指标(初期 = 键盘按键数)在赛道上竞速。基调是轻松的派对游戏感,是"带吉祥物的工具",不是游戏引擎项目。

## 架构不变量(不可违反)

1. **赛道 = SVG `<path>`。** 赛车位置永远用
   `getPointAtLength((pct/100) * getTotalLength())` 计算。
   换任何赛道形状 = 只换 path 的 `d` 属性,**绝不**硬编码赛车坐标,不动赛车/进度逻辑。

2. **UI 与外壳解耦,方向不可反。**
   - `index.html` = 纯 Web(无框架、无构建步骤),托管在 GitHub Pages,任意设备可改。
   - Tauri 外壳只**加载** Pages 的 URL;在本地按系统构建。
   - 外壳不承载 UI 逻辑;UI 不依赖外壳才能在浏览器里跑。

3. **原生边界。**
   - 全局键盘统计**只在 Rust 层**做,webview **永不**直接读全局键盘。
   - webview 只通过 Tauri 事件 `keycount` **接收**计数,再映射到赛道位置。
   - 隐私:**只对按键计数**,绝不记录是哪个键、绝不记录任何内容。

4. **后端(原型阶段)。**
   一台设备当主机跑一个小 WebSocket 服务;客户端只上报**自己的** count;
   主机汇总并广播房间状态。日后把"主机"整体换成 PartyKit(云),**UI 逻辑一行不改**。

## 平台规则

- **Windows 是当前主力开发/构建目标**,macOS 之后支持。
- **Tauri 不跨平台编译**:Windows 包在 Windows 上构建,macOS 包在 macOS 上构建(或用 CI)。一套源码,两个二进制。
- UI 层(网页)跨平台免费,两端渲染一致。
- macOS 额外项(以后处理):透明需 `macOSPrivateApi: true`(影响 App Store 上架);全局输入需"输入监控"授权;rdev 在 mac 上可能需拆成独立进程。

## 技术栈(暂定,改动需说明理由)

纯 HTML/CSS/vanilla JS · SVG 渲染 · Tauri v2(Rust)外壳 · `rdev` 全局按键 · `ws`(Node)原型后端 · GitHub Pages 托管。

## 给 Claude Code 的护栏

- **不要**给网页 UI 引入前端框架或构建步骤(必须保持 Pages 可直接编辑),除非显式批准。
- **不要**把键盘统计逻辑搬进 webview。
- **不要**现在就上云后端;原型阶段是"主机即服务器"。
- 守住 SVG-path 不变量;不要硬编码赛车坐标。
- 隐私红线:只计数,不记录按键内容。
